# 🔄 アップデート手順書

## ユーザーへの更新反映の仕組み

```
あなたがGitHubにファイルをアップロード
        ↓
GitHub Pagesが自動で新バージョンを配信
        ↓
ユーザーが次回アプリを開いたとき
「新しいバージョンがあります」バナーが表示
        ↓
「今すぐ更新」を押すと即反映
```

---

## アップデート手順（毎回やること）

### Step 1: sw.js のバージョン番号を上げる

`sw.js` の1行目を変更する：

```js
// 変更前
const VERSION = 'v1.0.1';

// 変更後（数字を1つ上げる）
const VERSION = 'v1.0.2';
```

※ これをしないと古いキャッシュが残り、更新が反映されない

---

### Step 2: GitHubにファイルをアップロード

1. https://github.com/あなたのusername/juku-schedule を開く
2. 更新したいファイルをクリック
3. 右上の「鉛筆アイコン（Edit）」をクリック
4. 内容を変更
5. 「Commit changes」を押す

または、ファイルごとドラッグ＆ドロップでまとめてアップロードも可能。

---

### Step 3: 確認

数分後にアプリを開き直すと、ユーザー画面に
「🆕 新しいバージョンがあります！」のバナーが表示される。

---

## よくある質問

**Q: バナーが出ないユーザーがいる**
A: 一度アプリを完全に閉じて開き直すと出ます。
　 またはブラウザのキャッシュをクリアしてください。

**Q: 更新を強制したい**
A: ユーザーに「アプリを一度閉じて開き直してください」と伝えるだけでOKです。

**Q: バージョンを上げ忘れた**
A: sw.js のVERSIONを上げて再アップロードすればOKです。
